# Sample1 Beta E2E Stress Run - 2026-06-28

This note records the local beta stress result used before the `0.2.0-beta`
publication check. It is engineering evidence for the Spec2Test beta package,
not a certification result, not a product guarantee, and not an acceptance
target for user projects. The coverage target remains 100%.

## Scope

- Sample: complex C stress case corresponding to `samples/c-input04-complex/sample1.c`.
- Workflow: PerfectOne Docker KLEE/LLVM/MC/DC baseline, followed by Windows
  local LLVM/lld-link Coding Agent residual coverage.
- Final gate: `perfectone_validate_c_final_evidence`.
- Residual policy: retry while coverage can still grow, stop after two
  consecutive measured no-increase attempts, hard limit 10 attempts.

The final gate accepted the result because all required residual evidence was
present: generated artifact keys, measurement artifact keys, aggregate attempt
history, per-function residual classification, and the two-attempt plateau.

## Remaining Limit Classifications

- `TD_main_0_0`: crash-risk and infeasible-path evidence. The local variable
  assignment makes one branch structurally unreachable, and direct execution
  exposes pointer-bearing uninitialized structure risk.
- `func3`: max-coverage/infeasible-path evidence. The residual attempts did not
  increase coverage after the bounded formula paths were exercised.

These classifications are not a claim that the source is correct. They are a
record of why the beta verifier stopped below 100% on this stress sample.
